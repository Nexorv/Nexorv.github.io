PulseEvents Documentation

Events • Queue System • Commands • Permissions • Configuration

📦 Overview

PulseEvents is a modular Minecraft (Bukkit/Spigot) plugin that automatically runs server events with announcements, UI feedback and configurable rewards.

The system is built around stability, anti-abuse protection and a queue-based event execution model.

⚙ Installation

1. Download the plugin

2. Place it in plugins/

3. Restart the server

4. Configure /plugins/PulseEvents/config.yml

🎯 Event System

  • Each event is managed by EventManager
  • Only one event can run at a time
  • Events can be started manually or via queue
  • Announcement system runs countdown before start

📦 Event Queue System

PulseEvents includes a FIFO queue system for events.

  • Events are executed in order
  • Next event starts automatically after current ends
  • No overlapping events allowed
/pe queue add <event> — Add event to queue
/pe queue list — Show queue
/pe queue remove <index> — Remove from queue
/pe queue clear — Clear queue

🧠 Cooldown System

Cooldowns are fully configurable in config.yml.


cooldown:
  event-start: 300
  command: 5
  • Prevents event spam
  • Works after /reload
  • Supports command + event cooldown separation

⚙ Commands

/pe start — Start event
/pe stop — Stop event
/pe status — System status
/pe reload — Reload config safely
/pe queue add <event> — Add to queue
/pe queue list — View queue

🔐 Permissions

pulseevents.admin

pulseevents.reload

🛡 Anti-Abuse System

  • Single event lock (no overlapping events)
  • Command cooldown system
  • Event cooldown from config
  • Safe reload handling

⚡ Features

  • Automatic event system
  • Queue-based execution
  • BossBar / ActionBar / Title support
  • Economy rewards (Vault optional)
  • Custom item rewards (e.g. Coin Rain)

🔌 Compatibility

  • Bukkit
  • Spigot
  • Paper

Optional: Vault, PlaceholderAPI

⚠ Notes
