📦 Overview

PulseEvents is a modular Minecraft plugin designed for fully automated event execution with a structured lifecycle, queue system, and real-time UI feedback.

It is built with scalability and stability in mind, separating responsibilities into dedicated managers.

🏗 Architecture

  • EventManager — controls event lifecycle (start, stop, state)
  • AnnouncementManager — countdowns, broadcasts, pre-start messages
  • LiveUIManager — BossBar, ActionBar, Titles

Each component is isolated to prevent system conflicts and improve maintainability.

⚙ Installation

  1. Download the plugin (when available)
  2. Place it inside plugins/
  3. Start or restart the server
  4. Edit plugins/PulseEvents/config.yml

🎯 Event Lifecycle

  • Idle → Waiting for trigger
  • Announcing → Countdown active
  • Running → Event logic executing
  • Ending → Cleanup phase

Only one event can exist in "Running" state at any time.

📦 Queue System

PulseEvents uses a FIFO queue to manage events.

  • Prevents overlapping events
  • Automatically starts next event
  • Supports manual queue control
/pe queue add <event> — enqueue
/pe queue list — show queue
/pe queue remove <index> — remove
/pe queue clear — reset

🧠 Cooldown System

cooldown:
  event-start: 300
  command: 5
  • Global event delay
  • Command spam protection
  • Persists after reload

📡 Live UI System

  • BossBar countdown before event start
  • ActionBar updates during event
  • Title messages for key phases
ui:
  bossbar: true
  actionbar: true
  title: true

⚙ Commands

/pe start — force start
/pe stop — force stop
/pe status — current state
/pe reload — safe reload
/pe update — check updates
